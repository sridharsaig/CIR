Height Estimator using Differentiable Rendering

This story follows up the previous guide on Differentiable Rendering — [https://drive.google.com/file/d/1IKjfVeA5MaV38tqZrHBLHN3BljUfm_i6/view?usp=sharing].

In this blog, I focus discussing the development of my project being built upon Facebook AI Research — Pytorch3D Differentiable Renderer to estimate the height of an entity. I discuss my first level entry to start with Github projects like these, so as to aid an entry level programmer and how one could aim to move further. This is a project being developed in QLAB (Makerspace Lab), Queen’s University Belfast as a part of coursework.

[Facebook’s Pytorch3D is a github project that aims to integrate smoothly with deep learning methods for predicting and manipulating 3D data. It’s been used previously to empower research projects like Mesh R-CNN. Henceforth, this state of the art Differentiable Renderer is chosen towards working on the potential Height Estimator.]

Height Estimator is a potential project that aims towards extracting the height of a structure in 3D Space. This is achieved through obtaining the camera view position of the structure using Pytorch3D Camera Position Optimizer & processing it through a 3D Toolkit like Blender. 

I have initially experimented with Pytorch3D Differentiable Renderer to plan the project all thanks to my lecturer. I recommend reading the aforementioned guide of this project to understand Differentiable Rendering since it’s rare to find a comprehensive guide on learning the same. After scripting the guide, I began to accustom to an Ubuntu Operating System connected to a built-in high-end Nvidia GPU ( CUDA - Enabled ) to run the notebooks of Pytorch3D. [I do not recommend using Google Colab to work with Pytorch3D. I have misspent an entire day running the codes taking hours of time in Google Colab which in the end has turned out to be in vain. The same codes with the system setup took 5 minutes to run effortlessly. Hence, I suggest using personal GPUs instead of Google Colaboratory]. To begin with experimenting with a Github project like Pytorch3D one does not require advanced knowledge. I emphasize getting used to Command Line Interface (CLI) by learning terminal through this video in 10 min— https://www.youtube.com/watch?v=vhZLTp6N4XA. Then taking a detour towards w3schools to learn basic python along with File Handling will help get one get started to begin testing the source codes. This is an itinerary of three important aspects by my lecturer -

  • Understanding a concept in general
  • Availing the academic source codes 
  • Modifying them into a real-world project

Facebook’s Pytorch3D consisted of 4 projects namely — Deforming a sphere into Dolphin Mesh, Bundle Adjustment, Rendered Texture Meshes, Camera Position Optimization. I began by searching the installation requirements in the Git. INSTALL.md is the code file for Pytorch3D. Installation is done through Anaconda, a package manager which is thoroughly explained through a Google Scholar in this blog — https://towardsdatascience.com/which-python-package-manager-should-you-use-d0fd0789a250 and then the installation is shown through a Youtuber over here — https://www.youtube.com/watch?v=i8w3-DQibQY . The overall requirements stated by the Git are installed & from here not just me but one can test all the 4 projects by importing the particular modules. As you import the modules, the subsequent steps are blocked out into 3 boxes — Loading the necessary object, Optimizing the process, and Visualizing the output. This gives the outlook of availing the state of the art Differentiable Renderer. From here, running the code through Terminal to successfully outputting results, I’ve encountered various errors throughout the process. Every step of this process is a simple Git install accompanying Package Management and Importing necessary modules. However, checking compatibility of the process, understanding the implementation of the codes at necessary times, and choosing right versions of the imports is crucial for successful running of the code. Mainly throughout this process, importing right version of the CUDA Toolkit & enabling CUDA is the important aspect of all. Github conversations helped a lot aside from Stackoverflow in this journey to solve various problems like the one above that people already once faced in the past.

When it came to modification, it was the tough. I was advised Camera Position Optimizer out of the four to be the best project to be improved into a real-world accessory. Camera Position Optimizer visualizes the 3D Camera Position of a reference image using Differentiable Rendering. This outputs a propagation from the starting position of the image to the camera position given by the Differentiable Renderer through a GIF Image. In the beginning, it was perplexing to find a use of this process. I ran into testing if the given process can be used for any arbitrary obj file. Turbosquid is a professional site to find various 3D Models of objects in obj format. I have customized the Camera Optimization Loop to include any arbitrary obj file given the folder path. Fetching a 3D Soccer Ball Object from Turbosquid, I ran the code to output the back-propagation pipeline to output the soccer ball GIF. It successfully ran the process, however the limitation is the size of the object. If the size of the object increases massively, then the distance between the camera and object also increases outputting a scene with the object looking like a man from an airplane view. However, this isn’t a real useful application of the process in real-world. Hence, this is stated a fiasco for the time spent. It was just too minute for modification and application. This is a real despair of my time. At this point of despair, came the advised Height Estimator project as an applicable plan of action. This is a real-world project with a greater application. Crime investigation uses CCTV Footage to assess a crime scene. In such a scene, often estimating the height of entities helps narrow down various scenarios. It not only evidences the caught criminal but also recording the dimensions of entities gives rise to valuable 3D assets. This can be achieved as mentioned earlier, by first locating the CCTV camera. For instance, the above code outputs the GIF Image by back-propagating from the referenced position to the determined camera position by the Differentiable Renderer at a distance. So, if the determined camera position is the position of the CCTV Camera, the camera properties can be used to reconstruct the scene in Blender. This is done through inputting the masked image of the entity whose height is to be determined and the 3D model of the entity through which the camera position is estimated. Thus the camera information is in turn used to project the footage onto the 3D scene through Blender. A conceptualized 3D cube/cuboid in the scene around the entity gives the absolute height of the entity.
